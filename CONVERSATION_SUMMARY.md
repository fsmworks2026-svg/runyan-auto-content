## 🛠️ 技術スタック選定の理由

## 🎓 重要な判断ポイント

### 「なぜ VTuber 的な 2次元推し活モデルか？」
1. **グレーな高額セールス誘導が不要** → 自然なファン化でいい
2. **顔出しなし** → 伏見さんの制約条件クリア
3. **完全自動化可能** → AI + AI の相性が良い
4. **推し活ファン層は課金意欲が高い** → サブスク向き

### 「段階的アプローチの意義」
1. **品質保証** → 毎回 Discord で確認（AI の失敗を防ぐ）
2. **データドリブン** → 何が受けるかを実測する
3. **リスク回避** → 一気に1,000本投稿して失敗するより、100本で学ぶ
4. **持続可能性** → 完全自動化でも「つまらない回」を除外できる

## 📌 VS Code 実装時の注意点

### ローカルテスト
```bash
# 1. 仮想環境アクティベート
source venv/bin/activate  # Mac/Linux

# 2. 環境変数確認
cat .env  # 4つのキーが設定されている？

# 3. 接続テスト
python test_connection.py

# 4. メイン実行
python main.py  # Discord に通知が来た？
```

### GitHub への push
```bash
# .env と credentials.json は絶対に push しない
# (.gitignore で保護されている)

git add -A
git commit -m "Initial implementation"
git push origin main
```

### GitHub Secrets 設定
- OPENAI_API_KEY
- ELEVENLABS_API_KEY
- DISCORD_WEBHOOK_URL
- GOOGLE_CREDENTIALS_JSON（JSON 全体）
