# 📋 改善前後の比較チートシート

## 🔄 シナリオ生成の進化

### 改善前
```json
{
    "title": "新しいカフェに行った",
    "scenario": "友達とカフェに行った話",
    "caption": "今日はこのカフェに行きました！",
    "mood": "happy",
    "setting": "カフェ",
    "makeup_style": "natural",
```

❌ 問題点:
- 「保存される理由」が不明確（カフェ情報がない）
- 「シェアされやすさ」を考慮していない
- 視聴者が「参加」できる要素がない
- アルゴリズムの「保存」「シェア」シグナルを無視

---

### 改善後
```json

    "title": "新大久保で穴場オシャレカフェを発見した話",

    // 🆕 新フィールド：エンゲージメント重視

    "share_element": "友達と一緒に行きたくなる雰囲気、予期しない発見",
    "interactive_element": "「このカフェ行きたい？」という投げかけ、画面をスワイプさせる",
    "engagement_signals": "保存・シェア促進のため、1枚目で場所を明示、キャプションで質問投げかけ",

```

✅ 改善点:
- 🎯 「保存される理由」が明確（場所・情報具体的）
- 📢 「シェアされやすさ」を設計（共感＆驚き）
- 🎪 インタラクティブ要素を組み込み（視聴者参加）
- 🤖 Instagramアルゴリズムのシグナル最適化

## ❓ よくある質問

### Q: 改善前の main.py は？
A: `main.py.bak` として バックアップしておきましょう。
   問題が出たら戻せます：`cp main.py.bak main.py`

### Q: シナリオが実装しなくなったら？
A: Claude API の model を確認してください：
   ```python
   model="gpt-4-turbo"  # これで問題なければOK
   ```

### Q: Discord 通知が表示されない新フィールドがある？
A: Claude がすべてのフィールドを返さないことがあります。
   その場合は main.py の 20行目付近の処理を確認。

### Q: 週別テーマを変更したい？
A: `generate_scenario()` の `theme_schedule` を編集：
   ```python
   theme_schedule = {
       0: "あなたが好きなテーマ",
       ...
   }
   ```
